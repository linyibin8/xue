# 知进伴学 · 「越用越聪明」智能体路线图（Goal + Loop 锚点）

> **总目标（Goal）**：让用户**只用自然语言**就能调用整个智能体的全部能力——提问、配置偏好、查看/调整上下文、增删记忆、消灭错题、扩充知识库——并且**每一轮问答都在滚动更新对用户的理解**，形成一个越用越懂他的「学习数字人」，由此建立粘性。
>
> 本文件是自驱动 loop 的终点定义与验收标准。loop 每轮推进一个增量（设计→编码→测试→发布→验收），直到三期全部 ✅。

## 角色（多角色交叉验证，互为校验）
- **PM/产品**：定义需求与验收标准，砍掉模糊功能。
- **交互设计**：界面与流程，确保用户「看得懂、能配置」。
- **iOS 架构**：iPad+iPhone 复用 AppState/组件，最小风险落地。
- **后端架构**：契约、上下文暴露、自然语言意图分流。
- **QA/对抗评审**：找漏洞/风险/自相矛盾/过度设计，质量闸门。
- **集成验收**：模拟器+真机+后端冒烟，对照验收标准签收。

## 三期目标与验收标准

### 一期：上下文可观测 + 设置精简（投入产出最高）
- **上下文/我的档案面板**：每轮问答清晰列出喂给模型的东西（问题、图片、本轮对话、偏好、命中长期记忆[带分]、错题、知识库、策略），每项**可开关本轮**、附「为何被调用」。两端一等公民。
- **设置精简**：砍掉重叠的 学习模式/讲解深度/学习策略 → 收敛为「一句话辅导偏好（自然语言）+ 2 个明确开关」；**学科自动识别**（不预设）；**恢复提示词配置入口**，与偏好统一为「辅导设置」。
- **验收**：① 任一轮问答可打开面板看到全部上下文项并逐项开关，开关即时影响下一轮；② 设置页只剩"偏好(NL)+开关+提示词"，无模糊下拉；③ 学科由内容自动打标；④ 两端一致；⑤ 回归不破坏现有问答。

### 二期：自然语言配置（NL → 调用智能体全部能力）
- **意图分流**：用户一句话若是「配置指令」（改偏好/删错题/记知识点/记住某事/调上下文）而非提问 → 识别意图 → 调对应能力 → 回执确认。后端已具 memory/mistakes/knowledge 写接口，补「意图识别 + 安全改写 + 确认」层。
- **app 内"管家"**：在对话里直接说"以后先给提示""把这道错题删了""记住我数学薄弱""新增知识点…"即可生效并可撤销。
- **验收**：① 10 条典型自然语言配置指令正确分流并生效+回执+可撤销；② 误分流率低（疑似配置但其实是提问 → 安全回退为提问）；③ 两端可用。

### 三期：可见的滚动记忆「数字人」
- **每轮问答后异步更新**画像/偏好/错题模式/已掌握知识，并**显式展示**「这次我更了解你了：…」，用户可查看、可纠正。
- **我的学习档案**：用户共同拥有、可编辑的数字人档案。
- **验收**：① 每轮后产生可见的记忆增量；② 用户可一键纠正/撤销某条记忆；③ 档案随使用稳定演进、检索可解释；④ 不污染（回放历史/配置指令不应误写记忆）。

## 工作流（每个增量都由「子智能体多角色」开发 + 交叉验证）
我（主控）做编排与集成；每个角色起一个**子智能体**，互相交叉验证：
1. **产品**子代理：读路线图+代码现状 → 需求与验收标准。
2. **交互**子代理：界面与流程设计（两端）。
3. **研发**子代理：按规格写代码（后端 surgical + AppState 共享核心 + iPad/iPhone 各接 UI）。多块独立时用 worktree 隔离并行，避免冲突。
4. **测试**子代理：独立验证（编译矩阵 + XCUITest + 后端冒烟 + 截图），不复用研发的判断。
5. **评审/对抗**子代理：审研发的 diff 与测试结论，找漏洞/回归/过度设计（交叉验证质量闸门）。
6. **验收**子代理：对照本期验收标准逐条签收。
7. 我集成：合并产出、处理 Mac 同步/构建/发布(bump-and-publish)、做最终判断。
> 原则：角色分工 + 交叉验证（研发≠测试≠评审，互不背书）；我负责协调与不可逆动作。
> **完全自主授权（用户 2026-06-25 重申）**：三期连续做完，含发布到真实 TestFlight，全程零停顿。**所有决定（含产品/设计取舍）由我自行评估拍板，不等用户**；遇分叉就选最符合总目标(越用越聪明)且最小风险的方案并记录理由。唯一例外=同一步骤反复失败且自身无法恢复(如签名/凭证彻底失效)，那时才报错求助。每期完成追加进度+memory，三期全 ✅ 后给唯一一次总报告。

## 增量2 · 拍照答题/批改大改造（2026-06-27 起，研究驱动，P0→P4）
用户实测反馈 v1「没做好」+ 两张截图(图糊/暗/低清，分割识别到「文字1处/矩形0个」)。多智能体调研(竞品交互+OCR模型+图像增强)结论：
- **根因=拍摄质量**，不是分割逻辑。后处理救不回没拍到的细节；严重模糊不可逆。**修拍摄端是最高杠杆**。
- **A vs B(让图变清晰)**：A=矫正+文档专用增强真实像素；B=大模型重生成数字版。**B 对批改是错的**(会幻觉/篡改学生真实作答、几何靠猜、丢证据)。**定论：整体走 A；B 仅限印刷题干可选重排版且强制对照原图，绝不用于学生作答/几何**。(用户仍想试 B→按此护栏做实验)
- **OCR：需要专用模型+大模型分工**(hybrid wins，2×4090 放得下)：服务端 dewarp+增强(DocRes) → 版面分题(PP-DocLayout/DocLayout-YOLO) → 印刷字 PP-OCRv5 + 公式 PP-FormulaNet 转写 → 喂 27B Qwen-VL 做几何理解/语义判分。几何无即插即用开源模型→交大模型。
- **交互对标**：小猿口算/作业帮=结果直接在原照片上叠 ✓/✗ 逐题，点错看讲解(消费已就位的后端 批改JSON)；取景框+边缘纠偏+CLAHE+全分辨率+清晰度门控+自动快门。分流：用户选「自动识别+一键纠偏」(intent_router+画面信号)。
